# MeshCore app assets

Branding assets for the MeshCore app — used by the Tanmatsu launcher when the
app is installed via the SD-card / app-store path (not via the badgelink
AppFS-only upload).

| File | Use |
|------|-----|
| `icon-32.png` | 32×32 launcher tile (the size the launcher's `app_metadata_parser` reads via `icon.32x32`). |
| `icon-256.png` | High-resolution variant for appstore listings, marketing materials, README screenshots. |
| `metadata.json` | App-store metadata: name, description, version, author, license, repo, icon mapping, target binaries. |

## Concept

The icon shows a 4-node mesh with an amber centre node — the visual cue for
"multi-node messaging" that's at the heart of MeshCore. Tokyo Night palette
matches the in-app theme (blue accent on deep indigo). Rounded square frame
follows launcher convention.

## Install flow that uses the icon

`badgelink appfs upload` (our `upload.sh`) uses only the AppFS metadata (slug,
title, version, size) — the launcher falls back to `ICON_APP` for those apps.

For the custom icon to show on the launcher tile:

1. Drop `application.bin` + `icon-32.png` + `metadata.json` in a folder named
   after the slug (`nl.cj.meshcore/`) inside the launcher's apps directory
   (SD card `/sd/apps/` or the configured install path).
2. The launcher's `create_list_of_apps_from_directory` reads `metadata.json`,
   loads the 32×32 PNG, and displays it in the tile.

Future appstore upload would use the same `metadata.json` schema.
